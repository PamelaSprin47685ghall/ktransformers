2.4 关键难点
权重数量和取值差异明晰后，我们参考llama.cpp代码，明确了权重回推的所有难点：

1. 权重命名映射: GGUF 与 HF 使用不同的命名规范
2. V Heads 重排序: HF→GGUF 转换时进行了 grouped → tiled 重排，需反向处理
3. in_proj_qkvz 合并: GGUF 中拆分为 attn_qkv + attn_gate，需合并回 in_proj_qkvz
4. Tensor 形状变换: 如 conv1d.weight 从 2D (6144,4) 需转换为 3D (6144,1,4)
5. 数值转换: A_log 等特殊权重需要进行数学变换
6. Norm 权重偏移: 部分 norm 权重存在 ±1 偏移
3.核心实现方案
根据以上分析，我们结合原始版本[6]权重做如下转换：

1. 加载原始 HF 模型作为基础模板
2. 加载主 GGUF 文件
3. 加载 mmproj GGUF 文件
4. 替换语言模型权重 (应用必要变换)
5. 替换视觉模型权重
6. 保留原始 MTP 权重
7. 保存输出 (支持分片)

进行HF→GGUF 转换时，V heads 做了 grouped → tiled 重排：

原始 (grouped):  [K0_V0, K0_V1, K1_V0, K1_V1, ...]  （按 K head 分组）GGUF (tiled):    [K0_V0, K1_V0, ..., K0_V1, K1_V1, ...]  （V 交错排列）
GGUF→HF 需要做反向：tiled → grouped，具体的就是一个reshape+permute操作：

# 输入 tensor 的目标维度: [num_v_per_k * num_k_heads * head_dim]# 1. reshape 为 [num_v_per_k, num_k_heads, head_dim]tensor = tensor.reshape(num_v_per_k, num_k_heads, head_dim)# 2. permute 交换前两维: [num_k_heads, num_v_per_k, head_dim]tensor = tensor.permute(1, 0, 2)# 3. 展平回原始形状: [num_k_heads * num_v_per_k * head_dim]tensor = tensor.reshape(original_shape)

原因：HF 中存储的是 A_log（对数形式），HF→GGUF 转换时做了 t = -exp(A_log)，即存储了负指数形式。反向需要 log(-t) 还原。

conv1d.weight：V reorder + 维度恢复
# 先在 2D 形状上做 V reorder（因为 reorder 沿 dim=0） t = apply_v_reorder_reverse(t, hf_name, config) # 再 reshape: (out_ch, kernel) → (out_ch, 1, kernel) t = t.reshape(out_ch, 1, kernel) 
原因：HF 中 Conv1d 权重是 3D (out_ch, 1, kernel_size)，GGUF 存储时压缩掉了 groups 维度变成 2D (out_ch, kernel_size)。顺序很关键——必须先做 V reorder 再 reshape，因为 reorder 操作的是 dim=0，reshape 后维度变了会导致错误。
