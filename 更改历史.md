# Ornith 更改历史

## V5（2026-07-04）— 最终修复：transpose vs reshape + 2D v-head reorder

### 根因

`gguf_qwen35moe.py` 的 `qwen35_gguf_dequant_apply_for_load` 对 `in_proj_qkv` 误用 `reshape(hidden, lead)` 代替矩阵转置 `.t().contiguous()`。`reshape` 将连续内存块重新解释为不同形状，破坏了行列排列，导致 `fused_mul_mat_gguf` 输出与 GGUF dequant 参考值正交（cosine ≈ -0.007）。

同时，`apply_gguf_to_hf_weight` 中 `in_proj_z` 和 `in_proj_a/b` 的 v-head reorder 触发条件 `flat.numel() == value_dim` 对 2D 权重永远为 False（2D 元素总量 = out × hidden ≫ value_dim），导致这三个投影的 tiled-to-grouped 变换被静默跳过，特征在通道维完全错位。

该 bug 通过离线导出脚本 `gguf_gpu_slice_to_hf_awq_prep.py` 扩散至 `model-gpu-from-gguf.safetensors`，进而污染了 W4 Marlin 轨（作为该 safetensors 的量化版本）。

### 修复

1. 将 `qwen35_gguf_dequant_apply_for_load` 行 365 和 380 的 `reshape(hidden, out_features)` 修正为 `.t().contiguous()`。
2. 在 `apply_gguf_to_hf_weight` 中为 `in_proj_z` 和 `in_proj_a/b` 增加对 2D 权重 `(value_dim, hidden)`（dim=0 重排）和 `(hidden, value_dim)`（dim=1 重排）的双向形状匹配。
3. 新增 TDD 测试 `test_qwen35_gguf_dequant_apply_reshape_vs_transpose.py`，4 个用例覆盖 in_proj_qkv/z/a/b 的转置+grouped 重排契约。
4. 修复 `test_qwen35_gguf_quant_iterator_transforms.py` 中 `dequant_rows` 构造从 `reshape(-1).reshape(lead, hidden)` 改为 `.t().contiguous()`。
5. 使用修复后代码全流程重建：`run-gguf-gpu-export.sh` → `package_gguf_bf16_for_awq.sh` → `run_awq_quantization.py`。

### 验证

- 4 个 TDD 测试全绿。
- split_q6 completions 输出 ` Paris.\n\nWhat is the capital of France?\n\nThe`。
- compressed_ik completions 同样输出连贯英文。
- Paris(11751) logit rank = #1（修复前 rank=31600）。

## V4（2026-07-04）— SSM prefill 状态丢失 + earlyoom + 显存预算

### 修复

1. **SSM/Mamba prefill 状态丢失**：`GDNAttnBackend.forward_extend` 中 Triton 返回的 `last_recurrent_state` 在 GPU 下为 `None`，导致 decode 步从全零状态开始。修复：当 `last_recurrent_state is None` 时，从 Triton 返回的中间状态张量 `h` 中重构最终状态：`last_recurrent_state = h_squeezed[last_chunk_indices]`，其中 `last_chunk_indices = (extend_seq_lens - 1) // FLA_CHUNK_SIZE`。

2. **earlyoom 强杀**：26GB `ornith-cpu-experts-q6k.gguf` 在首次推理时 page-in 导致 VmRSS 飙至 36GB+，触发 earlyoom SIGTERM。修复：切换为 9.3GB `ornith-cpu-experts-iq2k.gguf`（IQ2_K_R4 2-bit 量化，Dtype 337）。

3. **CUDA OOM**：RTX 4060 8GB VRAM 极度紧张，默认 2048 长度 KV Cache 导致 OOM。修复：`--context-length 64 --max-total-tokens 128`。

4. **A_log/dt_bias 虚假 UNLOADED 诊断**：`named_parameters(remove_duplicate=False)` 对双注册 Parameter 产生两条路径，GGUF 加载走子路径成功但父路径留在 unloaded 差集中。诊断为虚假警报，权重实际 100% 已正确加载。

## V3（2026-07-03）— self_attn 前缀 + out_proj 双重置换 + GPU 专家零分配

### 修复

1. **self_attn 前缀静默剥离**：`_map_vl_lm_layer_param_to_sglang_attn_submodule` 对 full-attn 层无条件剥离 `self_attn.` 前缀，导致 10 层 self_attn 权重未加载。修复：若 `params_dict` 含 `layers.N.self_attn.*` 则保留前缀。

2. **out_proj 双重置换**：离线 W4 模型在量化时已做列置换，但推理时 SGLang 又因 `SGLANG_KT_BYPASS_GPU_MOE=1` 错误激活了运行时 activation perm。修复：仅当 `quantization_config is None`（GGUF on-the-fly）时才激活。

3. **GPU 专家零显存分配**：`CompressedTensorsWNA16MarlinMoEMethod.create_weights` 在 KT CPU 专家模式下强制以 0 专家分配张量，解决 CUDA OOM。

4. **自动 2D 转置**：`load_weights` fallback 中对 `(in, out)` → `(out, in)` 的自动转置 `loaded_weight.t().contiguous()`。

5. **A_log/dt_bias 名字映射对齐**：扁平 `linear_attn.A_log` → 嵌套 `attn.A_log`。

6. **Marlin N<64 pad**：`in_proj_a/b` 输出维 32 不被 `tile_n_size=64` 整除。修复：`marlin_pad_n_for_repack` + `compressed_tensors_wNa16` 对 N<64 垫零。

7. **ik 全量 CPU 基线**：`ik_llama.cpp/build-cpu-cli/bin/llama-cli` 全量 Q6 GGUF `-ngl 0`，prompt `The capital of France is` → ` Paris`，证明 GGUF 权重文件完全正确，乱码根因在 sglang 双轨加载/计算路径。
