# Ornith-1.0-35B 双轨推理最终报告

## 0. 一句话

Qwen3.5 混合架构（30 层 GatedDeltaNet + 10 层 self-attn + 256 专家 MoE）在 8GB GPU + 64GB RAM 消费级硬件上实现双轨推理：split_q6（GGUF on-the-fly，~2GB GPU）与 compressed_ik（W4 Marlin，~3.6GB GPU），CPU 专家走 kt-kernel LLAMAFILE（IQ2_K_R4 2-bit 量化，9.3GB）。E2E 输出 ` Paris.\n\nWhat is the capital of France?\n\nThe`，完全对齐 ik 全量 CPU 基线。

| `num_expert_shared` | 1 |

### 1.2 双轨分工

| 轨 | GPU 权重 | CPU 权重 | 量化 |
|----|----------|----------|------|
| split_q6 | `ornith-gpu-non-expert.gguf` ~2GB Q6_K on-the-fly dequant→BF16 | `ornith-cpu-experts-iq2k.gguf` 9.3GB IQ2_K_R4 | GPU 全 Q6_K；CPU 2-bit |
| compressed_ik | `ornith-gpu-w4-q6-parity-from-gguf` ~3.6GB W4 Marlin+BF16 | 同上 | GPU 非专家 W4 packed；CPU 2-bit |

MoE 路由专家（`ffn_*_exps`）完全由 kt-kernel `LlamafileMoEWrapper` 通过 `GGUFLoader` mmap 管理（CPU RAM，保持打包格式，forward 时按需 dequant 单个 expert）。SGLang 的 `gguf_quant_weights_iterator` 跳过这些张量防双重加载。

### 1.3 GDN 内部投影层

| 投影 | GGUF 名 | out | in | v-head reorder |
|------|---------|-----|-----|----------------|
| in_proj_qkv | `blk.N.attn_qkv.weight` | 8192 | 2048 | dim=0 仅 V 段 |
| in_proj_z | `blk.N.attn_gate.weight` | 4096 | 2048 | dim=0 整体 |
| in_proj_a | `blk.N.ssm_alpha.weight` | 32 | 2048 | dim=0 head_dim=1 |
| in_proj_b | `blk.N.ssm_beta.weight` | 32 | 2048 | dim=0 head_dim=1 |
| out_proj | `blk.N.ssm_out.weight` | 2048 | 4096 | 运行时 activation perm |
| conv1d | `blk.N.ssm_conv1d.weight` | 8192 | 4 | dim=0 仅 V 段 + unsqueeze(1) |
| A_log | `blk.N.ssm_a.weight`（F32） | 32 | — | grouped + `log(-x)` |
| dt_bias | `blk.N.ssm_dt.bias`（F32） | 32 | — | grouped，直通 |
| norm | `blk.N.ssm_norm.weight`（F32） | 2048 | — | 不减一（SSM 内部 GroupNorm） |
| input_layernorm | `blk.N.attn_norm.weight`（F32） | 2048 | — | `w - 1.0`（GemmaRMSNorm） |

### 1.4 v-head tiled→grouped 重排

| 参数 | 值 |
|------|-----|
| `v_per_k`（vpk） | 2 |
| `k_heads`（kh） | 16 |
| `head_v_dim`（hvd） | 128 |
| tiled 排列 | `(vpk, kh, hvd)` = `(2, 16, 128)` |
| grouped 排列 | `(kh, vpk, hvd)` = `(16, 2, 128)` |
| dim=0 permute | `reshape(vpk, kh, hvd, tail).permute(1, 0, 2, 3)` |
| dim=1 permute | `reshape(tail, vpk, kh, hvd).permute(0, 2, 1, 3)` |
| in_proj_a/b hvd | 1 |
| out_proj activation perm | `arange(4096).reshape(2,16,128).permute(1,0,2).flatten()` |

### 1.5 F32 变换链

| 张量 | 变换 |
|------|------|
| attn_norm / post_attention_layernorm / model.norm | `w - 1.0` |
| ssm_a → A_log | `log(-w)` + grouped reorder |
| ssm_dt.bias → dt_bias | 直通 + grouped reorder |
| ssm_conv1d.weight | V 段 grouped + `unsqueeze(1)` |
| linear_attn.norm（ssm_norm） | 不减一 |

## 3. 双轨起服参数

### 3.1 split_q6（GGUF on-the-fly）

```

--load-format gguf --quantization gguf --dtype auto
```

## 4. 全流程构建

### 4.1 权重链

```
ornith-1.0-35b-Q6_K-MTP-final.gguf (28GB)
  ├─ extract-gpu-non-expert.sh → ornith-gpu-non-expert.gguf (~2GB)
  │     └─ run-gguf-gpu-export.sh → model-gpu-from-gguf.safetensors (~4.7GB BF16, 630 tensors)
  │           └─ package_gguf_bf16_for_awq.sh → ornith-gpu-bf16-standalone/
  │                 └─ run_awq_quantization.py → ornith-gpu-w4-q6-parity-from-gguf/ (~3.6GB W4)
  └─ reextract-cpu-experts.sh → ornith-cpu-experts-iq2k.gguf (9.3GB)
```

### 4.2 环境变量

```
PYTHONPATH=sglang/python:ktransformers/kt-kernel/python
LD_LIBRARY_PATH=ik_llama.cpp/build-cpu-cli/src:.../ggml/src
SGLANG_KT_BYPASS_GPU_MOE=1
SGLANG_DISABLE_CUDNN_CHECK=1
CPU_GGUF=ornith-cpu-experts-iq2k.gguf  # 或 KT_WEIGHT=...
```

### 4.3 构建 kt-kernel

```bash
cd ktransformers/kt-kernel
CPUINFER_USE_CUDA=0 CPUINFER_CPU_INSTRUCT=AVX2 CPUINFER_FORCE_REBUILD=1 \
  uv pip install --python $ORNITH_ROOT/.venv-public-py312/bin/python -e .
```

### 4.4 起服

```bash
# split_q6（推荐，~2GB GPU）
CPU_GGUF=ornith-cpu-experts-iq2k.gguf \
  bash ktransformers/kt-kernel/scripts/ornith/run_sglang_split_q6.sh

# compressed_ik（W4 Marlin，~3.6GB GPU）
KT_WEIGHT=ornith-cpu-experts-iq2k.gguf \
  bash ktransformers/kt-kernel/scripts/ornith/run_sglang_compressed_ik.sh
```

## 9. 约束

- 禁止下载 ~70GB 官方 BF16 safetensors；权重来源于单文件 Q6 GGUF 导出链。
- 三个公共 fork 是唯一代码归属点，禁止 patch site-packages。
- CPU 指令集仅 AVX2，禁止 AVX512 污染，llama.cpp 仅 CPU-only 编译。
- MoE 专家权重单一来源：仅 kt GGUFLoader mmap。
- out_proj Q6_K 不做权重列 perm，靠运行时 activation perm。
- in_proj_z/a/b 的 v-head reorder 必须同时支持 `(out, in)` 和 `(in, out)` 两种 2D 形状。
